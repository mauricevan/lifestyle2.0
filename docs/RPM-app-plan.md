# RPM (Recover, Program, Monitor) — Product Plan

This document is the source of truth for RPM Phase 1 MVP implementation.

See the full product specification in the project kickoff document. Key principles:

- **Zero-effort UX:** no manual input unless unavoidable
- **One screen, one glance:** RPM gauge is the primary UI
- **System language:** flight-engine tone, never guilt-inducing copy
- **Phase 1 scope:** HealthKit + Health Connect, EP engine, Supabase sync, hyperfocus notifications (no laptop lock)

Implementation phases:

1. **Phase 1 (MVP):** baseline, gauge, hyperfocus push, evening TTS, onboarding
2. **Phase 2:** companion app, screen dim/lock, configurable thresholds
3. **Phase 3:** forest mode, geofencing, HRV voice AI
4. **Phase 4:** history dashboard, wearable adapters, auto-scaling baseline
